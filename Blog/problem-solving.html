Another problem I encountered was variables not being recognised. The problem solving technique I initially used was trying to was rewriting the code in a different way which also did not work. I then read the error message which made me aware that the variable name was unrecognised. I did some more online research and found that JS variables are block scoped, and that I defined the variable outside of the block I was trying to use it in, which I have now learnt and will apply in the future.

I feel quite confident writing pseudocode, especially after extensive practice during the javascript cafe challenge. I believe that pre-planning the functionality of the code really helps streamline the process and helps reduce the potential for error in future implementation fo the actual codebase. I feel confident now trying new things to try figure out what is wrong with the code. I especially enjoy putting console.logs at places to see if that section of the code is being run or working properly. I have also gotten used to reading error messages and have become better at interpreting them. I think currently my favourite technique is googling the problem and finding previous examples from sites such as stack overflow. I believe that this is currently the most effective method for me. Lastly, I have not had to ask me peers or coaches for help up to this point in the course, but I know I will have to during the bootcamp. Thus, I have prepared and am very willing to ask for help when it is required.
